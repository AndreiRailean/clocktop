# Rule: Git Commit Format

Whenever you write a git commit message for this project, you **MUST** follow the Conventional Commits specification.

## Format
`<type>(<scope>): <description>`

## Allowed Types
- **feat**: A new feature
- **fix**: A bug fix
- **test**: Adding or correcting tests
- **ci**: Changes to CI/CD config files and scripts
- **docs**: Documentation-only changes
- **style**: Formatting/styling changes that do not affect code semantics
- **refactor**: Code refactoring changes

## Examples
- `feat: implement unit testing framework`
- `docs: add development guidelines`
- `ci: configure github action workflow`
