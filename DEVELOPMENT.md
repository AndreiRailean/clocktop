# Development Guidelines

## Git Commits

All git commit messages in this repository must follow the [Conventional Commits](https://www.conventionalcommits.org/) specification.

### Structure
```
<type>(<scope>): <description>

[optional body]

[optional footer(s)]
```

### Common Types
- `feat`: A new feature
- `fix`: A bug fix
- `test`: Adding missing tests or correcting existing tests
- `ci`: Changes to CI/CD configuration files and scripts (e.g., GitHub Actions workflows)
- `docs`: Documentation-only changes
- `style`: Changes that do not affect the meaning of the code (formatting, white-space, etc.)
- `refactor`: A code change that neither fixes a bug nor adds a feature
